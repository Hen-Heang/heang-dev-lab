Step 5: Fragments & Layout

Thymeleaf fragments let you write HTML once and reuse it everywhere — navbars, footers, modals, sidebars.

🔄 th:replace vs th:insert

These are the two ways to include a fragment. They look similar but behave differently:

th:replace 
<div th:replace="~{fragments/footer :: copyright}">
  placeholder
</div>


<p>© 2024 All rights reserved.</p>

th:replace = the tag you write disappears. Only the fragment content remains.

th:insert 
<div th:insert="~{fragments/footer :: copyright}">
  placeholder
</div>


<div>
  <p>© 2024 All rights reserved.</p>
</div>

th:insert = the tag you write stays as a wrapper around the fragment.

📐 Fragment Reference Syntax — 3 Forms


th:replace="~{learn/fragments/navbar :: navbar}"
                      ↑ file path               ↑ fragment name


th:replace="~{learn/fragments/navbar}"
  → uses the entire file body as the fragment


th:replace="~{:: myFragment}"
  → looks for th:fragment="myFragment" in the current page

🎯 Fragments with Parameters

You can pass values INTO a fragment — like a function with arguments:


<div th:fragment="alert(message, type)"
     th:class="'alert alert-' + ${type}">
    <span th:text="${message}"></span>
</div>


<div th:replace="~{alerts :: alert('Saved successfully!', 'success')}"></div>
<div th:replace="~{alerts :: alert('An error occurred.', 'danger')}"></div>

We will use this pattern in Week 3 (form validation) for error messages.
